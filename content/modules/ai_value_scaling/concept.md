# AI Value & Scaling

**AI value** is the business result an AI system produces that you can measure — not the fact
that it uses AI. **Scaling** is the careful path that takes a promising AI pilot and turns it
into something that reliably serves the whole organisation. (A pilot is a small first test.)
The two go together: you scale *because* a pilot proved real value, and you measure value *so
you know* whether to scale at all.

**Real value vs. vanity metrics.** A vanity metric looks impressive but is not tied to a
business result — "10,000 prompts run this month" or "the model is 92% accurate." A real value
metric is tied to money, time, risk, or quality: hours saved per analyst, support tickets
solved without a person, fewer errors, revenue influenced. Always ask: if this number doubled,
would the business be better off? If not, it is a vanity metric.

**The pilot → production → scale path.** AI work moves through stages:

- **Pilot** — a small test with a time limit, a clear success metric, and a real group of
  users.
- **Production** — the pilot becomes a dependable, watched system with support and owners.
  (Production means the real, live system that people depend on.)
- **Scale** — you extend it to more users, teams, or uses once value is proven.

**Deciding to scale or kill.** A pilot is an experiment, and many should be stopped. Scale when
the value metric passes a threshold you agreed on beforehand, real use is happening, and costs
are sustainable. **Kill** it when value is unproven, use is low despite effort, or it costs
more to run than it is worth. Killing a weak pilot is a success, not a failure — it frees
budget for better bets.

**Change management & adoption.** Technology that no one uses creates zero value. (Change
management is the work of helping people move to a new way of working.) Adoption depends on
training, trust, clear workflows, and showing people the AI helps them rather than threatens
them. Measure adoption (active users, repeat use), not just whether the tool is available.

**AI maturity.** Organisations grow from one-off experiments, to repeatable pilots, to
governed production systems, to AI woven into core operations. Maturity is about process and
people, not just models.

## How each role uses this

- **Portfolio Manager:** Owns the scale-or-kill decision and the budget shift, telling real value from vanity metrics when reporting to leadership.
- **Project Manager:** Defines the success metric and threshold up front, runs the pilot, and owns the adoption and change-management plan so people actually use the system.
- **Developer:** Instruments the system so real value metrics (time saved, error rate) and adoption are measured, and builds it to scale only once value is proven.
- **Enterprise Architect:** Checks the system is reliable enough for production, and designs it so cost and quality don't collapse as use grows.
