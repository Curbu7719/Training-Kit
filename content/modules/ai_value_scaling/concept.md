# AI Value & Scaling

**AI value** is the measurable business outcome an AI system produces — not the fact that it
uses AI. **Scaling** is the disciplined path that takes a promising AI pilot and turns it
into something that reliably serves the whole organisation. The two are inseparable: you
scale *because* a pilot proved real value, and you measure value *so you know* whether to
scale at all.

**Real value vs. vanity metrics.** A vanity metric looks impressive but does not tie to a
business outcome — "10,000 prompts run this month" or "the model is 92% accurate." A real
value metric connects to money, time, risk, or quality: hours saved per analyst, support
tickets resolved without a human, error rate reduced, revenue influenced. Always ask: if
this number doubled, would the business be better off? If not, it is a vanity metric.

**The pilot → production → scale path.** AI work moves through stages:

- **Pilot** — a small, time-boxed test with a clear success metric and a real user group.
- **Production** — the pilot becomes a dependable, monitored system with support and owners.
- **Scale** — you extend it to more users, teams, or use cases once value is proven.

**Deciding to scale or kill.** A pilot is an experiment, and many should be stopped. Scale
when the value metric clears a pre-agreed threshold, adoption is real, and costs are
sustainable. **Kill** when value is unproven, adoption is low despite effort, or the cost to
run it exceeds the benefit. Killing a weak pilot is a success, not a failure — it frees
budget for better bets.

**Change management & adoption.** Technology that no one uses creates zero value. Adoption
depends on training, trust, clear workflows, and showing people the AI helps rather than
threatens them. Measure adoption (active users, repeat use), not just availability.

**AI maturity.** Organisations progress from ad-hoc experiments, to repeatable pilots, to
governed production systems, to AI woven into core operations. Maturity is about process and
people, not just models.

## How each role uses this

- **Developer/Engineer:** Instruments the AI system so real value metrics (time saved, error
  rate) and adoption are measured, and builds it to scale only once value is proven.
- **Business Analyst:** Defines the success metric and threshold up front, and separates real
  value from vanity metrics when reporting results.
- **PM/Product Owner:** Owns the scale-or-kill decision, the adoption plan, and the change
  management needed for people to actually use the system.
- **QA & Architect:** Validates the system is reliable enough for production and designs it to
  scale without cost or quality collapsing as usage grows.
